# Politica PLACEHOLDER

Un `PLACEHOLDER` è una categoria tecnica, non un’asserzione archeologica. Deve avere `featureId`, `assetId`, ragione, requisito di sostituzione, stato M0, trasformazione di display e campi scientifici NULL o vuoti.

Sono vietati `confidenceGrade`, `epistemicKind`, `DEFINITIVE`, `RELEASED` e `releaseEligible=true`. Il validatore di build applica questi vincoli e verifica l’allowlist pubblica.
